# 清掃報告書アプリ セットアップ手順

初心者の方でも順番通りに進めれば完成できます。
所要時間：約 30〜45 分

---

## ステップ 1：Supabase のセットアップ

### 1-1. アカウント作成
1. https://supabase.com にアクセス
2. 「Start your project」→ GitHub でサインイン

### 1-2. 新しいプロジェクトを作成
1. 「New project」をクリック
2. 以下を入力：
   - **Project name**: `cleaning-report`（なんでもOK）
   - **Database Password**: 強いパスワードを設定（メモしておく）
   - **Region**: `Northeast Asia (Tokyo)` を選択
3. 「Create new project」→ 2〜3分待つ

### 1-3. データベースとストレージを設定
1. 左メニューの「**SQL Editor**」をクリック
2. 「New query」をクリック
3. プロジェクトフォルダの `supabase-schema.sql` ファイルを開き、
   中身をすべてコピーして貼り付け
4. 「**Run**」ボタンをクリック（エラーが出なければOK）

### 1-4. API キーをコピー
1. 左メニューの「**Project Settings**」→「**API**」
2. 以下の2つをコピーしてメモ：
   - `Project URL`（https://xxxxxxxx.supabase.co）
   - `anon` `public` のキー（eyJxxxxxxxx...）

---

## ステップ 2：アプリの設定

### 2-1. 環境変数ファイルを設定
プロジェクトフォルダ（`cleaning-report`）の中に
`.env.local` というファイルがあります（隠しファイル）。

**Mac でターミナルを開いて以下を実行：**
```bash
cd ~/Desktop/cleaning-report
open .env.local
```

テキストエディタで開いたら、以下のように書き換えて保存：
```
NEXT_PUBLIC_SUPABASE_URL=（ステップ1-4でコピーしたProject URL）
NEXT_PUBLIC_SUPABASE_ANON_KEY=（ステップ1-4でコピーしたanonキー）
```

---

## ステップ 3：アプリを起動する（ローカル確認）

ターミナルで以下を実行：
```bash
cd ~/Desktop/cleaning-report
npm run dev
```

ブラウザで http://localhost:3000 を開く
→ ログイン画面が表示されれば成功！

### 最初のアカウントを作成
1. 「新規登録」をクリック
2. メールアドレスとパスワードを入力して「アカウント作成」
3. Supabase の Inbucket（テスト用メール）で確認メールを受け取る
   - Supabase Dashboard → Authentication → Users で確認メールをスキップできます
   - 左メニュー Authentication → Users → ユーザーの「...」→「Confirm email」

---

## ステップ 4：インターネットに公開（Vercel）

iPhoneやPCどこからでもアクセスできるようにします。

### 4-1. GitHub にコードをアップロード
1. https://github.com でリポジトリを新規作成（名前：`cleaning-report`）
2. ターミナルで：
```bash
cd ~/Desktop/cleaning-report
git add .
git commit -m "初回コミット"
git remote add origin https://github.com/（あなたのユーザー名）/cleaning-report.git
git push -u origin main
```

### 4-2. Vercel にデプロイ
1. https://vercel.com にアクセス（GitHub でサインイン）
2. 「Add New Project」→ `cleaning-report` リポジトリを選択
3. 「Environment Variables」に以下を追加：
   - `NEXT_PUBLIC_SUPABASE_URL` = Supabase の Project URL
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` = Supabase の anon キー
4. 「Deploy」をクリック（2〜3分で完了）
5. 表示されたURL（https://cleaning-report-xxx.vercel.app）でアクセス可能に！

### 4-3. Supabase に本番URLを登録
1. Supabase Dashboard → Project Settings → API → URL Configuration
2. `Site URL` に Vercel の URL を入力して保存

---

## 使い方

### 基本的な流れ
1. **ログイン** → アカウントでログイン
2. **物件を登録**（「物件」タブ）→ 物件名・住所を入力
3. **撮影項目を設定** → 物件をタップ → 「玄関」「トイレ」など追加
4. **報告書を作成**（「報告書」タブ → 「新規作成」）
   - 物件を選択
   - 清掃日を入力
   - 各項目のビフォー・アフター写真を撮影/選択
   - 保存
5. **PDFで書き出す** → 報告書を開いて「PDF出力」

### iPhone での使い方
- Safari で Vercel の URL を開く
- 「共有」→「ホーム画面に追加」でアプリのように使える
- カメラボタンをタップするとiPhoneカメラが起動して直接撮影可能

## トラブルシューティング

| 問題 | 解決方法 |
|------|---------|
| ログインできない | Supabase Dashboard → Authentication → Users でメール確認済みにする |
| 写真がアップロードされない | .env.local のキーが正しいか確認 |
| PDFに写真が表示されない | しばらく待つ（外部URLからの読み込みに時間がかかることがある） |
| 「物件が見つかりません」 | Supabase の SQL Editor で schema を再実行 |

## 複数人で使う場合

チームメンバーは Vercel の URL からアクセスして「新規登録」するだけです。
全員が同じ Supabase プロジェクトのデータを共有します。
